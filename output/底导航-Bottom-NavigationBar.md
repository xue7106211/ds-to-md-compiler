## 一、概述

底部导航（`Bottom NavigationBar`）用于在应用内进行导航操作，通过点击每个 Tab 可前往应用不同的模块。

- 应用在**手机**上均采用底部导航。
- 在**大屏设备**上（Fold 折叠屏、平板）时，部分采用了「层级导航」的应用为了增加导航效率，会将底部导航转换为 `侧边导航栏 Sidebar`。

### 重要原则摘要

1. 底部导航的作用是让用户在 App 的不同模块之间切换，比如"时钟"应用中的"闹钟"、"时钟"、"秒表"、"计时" Tab。
2. 如果你需要可承载针对当前页面内项目的操作的空间，请使用 `底部工具栏 Menubar`。

## 1.1 适用的平台

- 手机
- Fold 折叠屏内屏
- 平板设备

## 二、组件构成

| 序号 | 主要构成 | 说明 |
| --- | --- | --- |
| ❶ | 图标 | 每个选项的图标 |
| ❷ | 文字 | 每个选项的标签文字 |

每个选项由 ❶ 图标 + ❷ 文字组成。

![底导航组件构成示意：标注图标与文字两个构成元素](图像占位符)

```yaml
visual_data:
  component: Bottom NavigationBar
  regions:
    - id: icon
      label: "❶图标"
      role: 各选项的视觉标识
    - id: label
      label: "❷文字"
      role: 各选项的文字标签
```

## 三、组件分类及交互规则

图标+文字有**竖排**、**横排**两种布局方式，将根据窗口宽度自适应：

- 从设计上，小屏用竖排，大屏空间充足时用横排。
- 仅靠「设备」判断横排还是竖排不够，需用「窗口宽度」判断。

### 布局切换阈值

| 条件 | 布局方式 |
| --- | --- |
| 窗口宽度 > `960` | 图标+文字 **横排** |
| 窗口宽度 < `960` | 图标+文字 **竖排** |

### 整体交互规则

- 最多可以放 `5` 项，最少 `2` 项。
- 支持页面内**左右滑动**切换 Tab。
- **点击**可切换 Tab。
- 若点击**当前 Tab**，当前页面自动回到顶部。

![竖排底导航示意](图像占位符)

![横排底导航示意](图像占位符)

```yaml
interaction_rules:
  layout:
    threshold_window_width: 960
    below_threshold: vertical
    above_threshold: horizontal
  items:
    min: 2
    max: 5
  selection:
    tap_to_switch: true
    swipe_content_to_switch: true
    tap_current_tab: scroll_to_top
```

## 六、易混淆组件辨析

### 1. 工具栏 vs 底部导航 vs 子页签

| 组件 | 作用 | 不可并存的情况 | 页面级别 |
| --- | --- | --- | --- |
| `底部工具栏 Menubar` | 通常用于在编辑模式下展示操作选项，是针对当前选中项的操作 | 因位置可能冲突，不可与「底导航」并存 | 所有 |
| `底导航 Bottom NavigationBar` | 对应用功能进行分流导航 | 不可与「工具栏」并存 | 一级 |
| `子页签 Tabs` | 对当前页面的内容进行分区 | / | 所有 |

### 2. 底部导航 vs 侧边导航

> 📌 底部导航和侧边导航栏本质上都是一种导航方式，一个应用中**不允许**既有底部导航，又有侧边导航。

| 组件 | PAD 端应用选用 |
| --- | --- |
| `底导航 Bottom NavigationBar` | `null` |
| `侧边导航栏 Sidebar` | `null` |
