## 一、概述

弹窗是操作系统和用户交互的窗口，主要用于向用户发起警示、提醒、确认等行为，并承担部分快速操作的职能。

模态弹窗具有最高操作优先级，要求用户必须在处理完当前弹窗后才能继续进行其他操作。为了避免过分打扰用户，建议避免过多地使用弹窗。

## 二、类型

| 序号 | 类型 | 定义 | 常见场景 |
| --- | --- | --- | --- |
| 1 | `确认弹窗` | 系统需要用户确认某些操作或信息 | 确认是否要执行操作（加密、停用功能等）；执行中系统推荐额外操作（推荐用 xxx 打开、推荐下载原图等）；告知出现 xx 问题、操作失败（无法配对、密码不正确等） |
| 2 | `警示弹窗` | 系统需要警示用户即将进行无法撤销的操作 | 删除某一文件/内容；退出后不会保存现有编辑内容；清除数据，还原系统 |
| 3 | `授权弹窗` | 要求用户确认哪些权限是允许系统获取的 | 必须同意才能使用的功能；初次打开应用时选择应用可获取的权限 |
| 4 | `引导弹窗` | 有大图的弹窗，通常用于功能、操作引导 | 功能升级 & 内容增多的推送；新手引导 |
| 5 | `输入弹窗` | — | — |
| 6 | `进度弹窗` | 向用户传达当前的加载状态 & 进度 | 加载中；更新中；传输 & 导入 & 下载中 |
| 7 | `其他弹窗` | — | 见 7.1、7.2 |

## 三、结构

通常一个弹窗由以下几个部分构成：

| 序号 | 区域 | 说明 |
| --- | --- | --- |
| 1 | 标题区域 | 图标 / 标题 |
| 2 | 内容区域 | 正文 / 列表 / 输入框 / 进度条 / 时间选择器 / CheckBox / 注释 |
| 3 | 自定义区域 | 需自行实现 + 评审 |
| 4 | 按钮区域 | 见 4.1、4.2 |

![弹窗四区域结构示意图：标题区、内容区、自定义区、按钮区](image-placeholder-dialog-structure)

```yaml
structure_dialog:
  regions:
    - id: title_area
      elements: [icon, title]
    - id: content_area
      elements: [body_text, list, input, progress_bar, picker, checkbox, annotation]
    - id: custom_area
      implementation: 需自行实现并评审
    - id: button_area
      layout_rules: 见按钮区域规则
```

### 3.1 标题区域

MIUIX 当前支持的控件种类包含图标 / 标题。

| 元素 | 规格 | 说明 |
| --- | --- | --- |
| `图标` | 可选 | 支持两种尺寸：`64dp`、`24dp` |
| `标题` | 必选 | 居中排布；原则上在中文场景的标准字体模式下标题建议两行以内（超出需优化文案）；MIUIX 中涉及国际化和大字的自适应规则是超 4 行和内容区（标题区 + 内容区 + 自定义区）一起滚动浏览，且操作区固定；标题内容遵从简洁明晰原则，尽量避免折行 |

### 3.2 内容区域

MIUIX 当前支持的控件种类包含正文 / 列表 / 输入框 / 进度条 / 时间选择器 / CheckBox / 注释。

| 元素 | 排布规则 |
| --- | --- |
| `正文` | 默认居中排布；折行时左对齐排布；避免中文文本下最后一行只有一个字的情况；避免标题与正文重复描述，若文案内容较为简单，可选择仅使用标题 |
| `备注文字` | 对齐形式与正文保持一致 |
| `CheckBox` | 若存在 CheckBox，则统一将正文和 CheckBox 居左对齐 |

### 3.3 自定义区域（需自行实现 + 评审）

自定义区域的可选元素包括但不限于：带开关的列表、分割线、备注字、插图等。

### 3.4 按钮区域（更新）

#### 4.1 基本布局

| 按钮数量 | 布局规则 |
| --- | --- |
| 单个按钮 | 居中，通常为普通按钮 |
| 多个按钮 | 弹窗按钮默认左右布局；当按钮文案在多语言大字体等情况下显示不完全时，从左右排布变为上下排布 |
| 两个以上按钮 | 采用上下排布方式 |

#### 4.2 多个按钮时排布规则

| 序号 | 规则 | 说明 |
| --- | --- | --- |
| 1 | 按钮顺序 | 优先保证顺序性，用户最有可能点击的放在纵列的最上方 / 横排的最右方 |
| 2 | 推荐按钮（可选） | 蓝色高亮的按钮为系统推荐，可放置在纵列的最上方或横排的最右方，不可在一组按钮的中间位置出现 |
| 3 | 正向操作 | 语义是表正向 / 继续操作类的操作不可用警告样式 |
| 4 | 负向操作 | 语义是表负向 / 终止 / 拒绝操作类的操作不可用高亮样式 |
| 5 | 冲突处理 | 如果方案和 2、3、4 条有任意一条冲突，建议按钮统一使用普通样式 |

![弹窗按钮区域布局示意：单按钮居中、双按钮左右/上下排布](image-placeholder-dialog-buttons)

```yaml
button_area_rules:
  single_button:
    alignment: center
    style: 通常为普通按钮
  multiple_buttons:
    default_layout: 左右排布
    fallback_layout: 上下排布
    trigger: 多语言大字体等情况下显示不完全
  order_priority:
    vertical: 纵列最上方
    horizontal: 横排最右方
  recommendation_button:
    style: 蓝色高亮
    position: 纵列最上方 或 横排最右方
    forbidden: 不可在中间位置
```

## 四、响应式变化规则

| 规则类型 | 说明 |
| --- | --- |
| **高度规则** | 弹窗最大高度为当前屏幕高度的 `80%`；当手机处于上下分屏或横屏状态时高度不受上述规则约束；若遇到在横屏情况下无法保证正常显示可配置为左右布局（MIUIX 支持） |
| **键盘弹起** | 当弹窗内存在需要弹起键盘以操作时，弹窗将与键盘固定 `16dp` 间距；当键盘收起时弹窗重新居底 |
| **宽度规则** | 弹窗所在窗口宽度 `< 360dp` 时约束弹窗距离屏幕左右边距为 `8dp`；弹窗所在窗口宽度 `>= 360dp` 时约束弹窗宽度固定为 `360dp` |

### 4.1 位置规则

| 设备 / 状态 | 水平位置 | 垂直位置 | 备注 |
| --- | --- | --- | --- |
| 手机 & 折叠屏 | 左右居中 | 上下居底 | 距小白条 & 虚拟键 `16dp` |
| 手机横屏 / 分屏 | — | 居底 | 距小白条 `16dp` |
| 平板 | 左右居中 | 上下居中 | — |
| 平板分屏 | 左右居中 | 上下居中 | — |

```yaml
responsive_dialog:
  height:
    max_percent: 80
    split_screen_landscape: 不受约束
  keyboard:
    spacing_dp: 16
  width:
    when_lt_360dp: 左右边距 8dp
    when_gte_360dp: 固定 360dp
  position:
    phone_fold: 左右居中, 上下居底, 距小白条/虚拟键 16dp
    phone_landscape_split: 居底, 距小白条 16dp
    tablet: 左右居中, 上下居中
```

## 五、交互规则

### 5.1 关闭操作

弹窗可通过以下 4 种方式关闭：

| 方式 | 可配置性 |
| --- | --- |
| 1. 返回手势 | 可依据情况自行配置是否可触发关闭 |
| 2. 上滑杆子 | 可依据情况自行配置是否可触发关闭 |
| 3. 点击取消按钮 | 固定 |
| 4. 弹窗外区域做任何操作（点击、滑动等） | 可依据情况自行配置是否可触发关闭 |

### 5.2 弹窗内的滑动操作

| 场景 | 规则 |
| --- | --- |
| 弹窗高度超过最大高度 | 固定标题栏和按钮，中间的内容区域为可滑动区域 |
| 手机横屏 + 左右排布 | 左侧内容侧和右侧按钮侧均可滑动 |

### 5.3 交互状态

按钮存在以下状态：`默认`、`强调`、`按压`、`不可点击`、`警示`。具体请见「按钮设计规范」。

## 六、注意事项

1. 尽量避免出现弹窗跳弹窗的情况。
2. 由于单选弹窗在职能上与近手弹窗重复，建议优先使用近手弹窗。若选项需要解释 / 单个选项文案过长 / 从近手弹窗中触发时则不应当使用近手弹窗。
3. 建议优先考虑将 UI 控件直接嵌入页面中（尤其在 pad 中）。
4. 确保取消 & 确认按钮的存在。
