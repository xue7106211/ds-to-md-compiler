## 一、背景

抽屉窗口一般用于**临时任务或并行任务**，是可灵活变化尺寸的轻量内容容器。

### 1. 适用的平台

- 手机
- Fold 折叠屏内屏
- 平板设备

### 2. 其他平台考量因素

- **Flip 折叠屏的外屏**：不适用。

## 二、组件构成

| 类别 | 主要构成 | 区域概述 |
| --- | --- | --- |
| 导航非吸顶样（大标题） | ❶ 滑动把手 | 位于窗口顶部，用于改变高度和滑动关闭窗口。 |

## 三、组件分类及交互规则

### 3.1 竖屏 - 模态抽屉（背景不可交互）

| 维度 | 内容 |
| --- | --- |
| **用途** | 用于临时任务；弹出后其他区域有遮罩，被遮罩区域原内容不可交互；**点击遮罩区域不可关闭窗口**。 |
| **典型场景** | 主体架构外的任务，进行完不可通过系统返回回退。例如桌面编辑中的设置、文件互传菜单等，可作为原复杂弹窗的替代者。 |
| **触发原则** | ① 必须是用户主动操作后的直接反馈（如分享、编辑），或无需点击默认即展开（如日历）。② 在内容区处于顶部时，整个窗口区域支持下拉关闭（手势优先内容区域纵向滑动）；处于非顶部时，把手+标题栏区域支持下拉关闭。③ 下拉把手或整体区域可改变窗口高度；高度分为**低、中（默认）、高**三个尺寸，不支持无极悬停。 |

#### 高度尺寸说明（模态抽屉）

| 尺寸 | 说明 |
| --- | --- |
| **低** | 仅用于并行任务，如查看小部件效果、AOD 效果；仅保留把手与标题栏。 |
| **中** | 默认弹出尺寸，具体见视觉部分。 |
| **高** | 具体见视觉部分；拉伸到最大高度后，内容过长需要滚动查看。 |

#### 各区域交互（模态抽屉）

| 区域 | 滑动 | 点击 |
| --- | --- | --- |
| ❶ 滑动把手 | 类半屏状态上滑切换为类全屏；类全屏状态向下滑关闭面板。 | 点击半屏面板横条所在区域无响应；点击类全屏面板向下箭头所在区域（向下箭头）关闭面板。 |
| ❷ 标题栏 | — | 左侧关闭、取消可选；标题始终居中。 |
| ❸ 内容区 | — | 用于展示关键内容或任务，可在不同大小尺寸间切换；支持界面跳转，跳转后框架保持不变，把手始终存在。 |

### 3.2 竖屏 - 非模态抽屉（背景可交互）

| 维度 | 内容 |
| --- | --- |
| **用途** | 用于并行任务；无遮罩区域，窗口内外皆可操作；窗口在 Z 轴上高于非窗口区域，调整窗口高度会影响其他区域内容展示效果。 |
| **典型场景** | 同屏下两个互相关联的任务同时进行。如日历（看日历和看日程互相配合）；如小部件中心编辑时对已添加的查看效果。 |
| **关闭与把手** | 抽屉始终存在时不可以关闭窗口；抽屉在中途被拉出时需增加关闭按钮。非模态时若窗口只有一档尺寸且不能关闭，可以不展示把手。 |
| **高度与分屏** | ① 拉伸高度不会改变窗口内内容的形态或多寡，只影响容器高度；并行任务时，被遮挡区域内容可根据抽屉高度变化形态。② 页面处于顶部时，收起热区为全部（手势优先内容区域纵向滑动）；非顶部时，收起热区为把手+标题栏。③ 分屏应用中，每屏都可并列使用抽屉窗口。④ 窗口高度同样遵循低、中（默认）、高三个尺寸，不支持无极悬停。 |

#### 各区域交互（非模态抽屉）

| 区域 | 说明 |
| --- | --- |
| ❶ 滑动把手 | 可选展示；位于窗口顶部，用于改变高度和关闭窗口；当非模态时窗口只有一档尺寸且不能关闭时，可以不展示把手。 |
| ❷ 标题栏 | 和全屏页面的标题栏一样，但不支持大字标题；标题可选择左对齐或中心对齐。 |
| ❸ 内容区 | 用于展示关键内容或任务，可在不同大小尺寸间切换，方便用户浏览内容。 |

### 3.3 横屏、折叠屏、Pad

| 终端 | 说明 |
| --- | --- |
| **横屏** | 暂不支持，目前强制竖屏。 |
| **折叠屏、Pad** | 默认为浮窗；小部件属于特殊适配。 |

## 五、特殊情况适配

### Pad 端与分屏

- **Pad 端左右分栏结构页面**（例如笔记）：模态窗口整屏水平居中。
- **分屏应用**：每屏都可并列使用抽屉窗口。

### 1. 多语言

- 标题区遇到多语言超长的处理办法（见多语言设计指南或源 PDF）。

### 2. 大字号

- 图标不放大；文本最大倍数为 `1.45x`。
