### 交互规则

- **出现时间**
  - 同一个气泡只能出现一次。
  - 每次出现 `5s`。
  - 若 5s 内用户滑动或点击其他区域，则气泡收起。
  - 若气泡展示大于 5s，气泡自动收起。
- **对齐**
  - 气泡出现在关联元素旁边。
  - 气泡尖与焦点元素中心对齐。
- **文案**
  - 气泡内容仅包含简短的描述性文字，尽量短。
  - 气泡内展示内容一定要完全展示，不能出现「……」。
  - 若展示不全，需要优化所展示内容。
- **方向**
  - 气泡尖支持 **6 个方向**：左上、左下、居中上、居中下、右上、右下。
- **适配**
  - 气泡适配 `light` / `dark` 模式。
- **极限**
  - 气泡包含内容具有极限值：最短、最长、最高。

## 四、引导气泡和提示条的区分

| 维度           | 引导气泡 | Banner / 提示条 |
|----------------|----------|------------------|
| 内容/场景      | 新功能、特殊功能提示 | 推荐功能服务、解释功能、提示状态 |
| 生命周期及消失 | 自动出现，使用对应功能或二次打开界面时消失 | 必须用户手动操作后消失；不消失但可收起 |
| 用户预期       | 预期外   | 临时场景预期外；长驻场景预期内 |
| 打断程度       | 中断任务，影响中 | 不中断任务，影响中 |
| 当前任务关系   | 有       | 有               |
| 是否要求被阅读 | 不阅读可能影响任务 | 不阅读可能影响任务 |
| 备注           | 一个对象只能用一次气泡 | 支持滚动隐藏；对时效性要求不强 |

## 五、组件视觉规则

### 结构概述

| 类型     | 区域概述 |
|----------|----------|
| 常规样式 | `①容器`：包含安全区域和信息内容。 |
|          | `②箭头`：与阐释内容中心点居中对齐。 |
|          | `③内容区`：填充文字内容；适配多语言；有最多和最少字段限制。 |
|          | `④阴影`：引导气泡具有很强的临时性，阴影需要辅助更加突出气泡内容。 |

### 容器

| 属性     | 规范 |
|----------|------|
| 安全区域 | 上下 `15dp`、左右 `14dp` |
| 边距     | 极左和极右的最小边距为 `24dp` |

### 箭头

| 属性       | 值    |
|------------|------|
| 顶端圆角   | `2dp` |
| 折角圆角   | `20dp` |
| 顶端角度   | `90°` |

### 阴影

| 属性  | 值 |
|-------|-----|
| Blur  | `35dp` |
| Y     | `17` |
| X     | `0` |
| Alpha | `#000000` `10%` |

```yaml
shadow:
  blur_dp: 35
  offset:
    x: 0
    y: 17
  color: "#000000"
  alpha: "10%"
```
