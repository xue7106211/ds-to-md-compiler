# 信息提示 Notice Bar

## 一、概述

信息提示用于展示系统的重要信息，同时用于引起用户关注又不打断用户当前浏览或操作的组件。

### 重要原则摘要

1. 只有文字的提示条不支持交互；需要交互的提示条需配备右箭头、关闭、操作按钮。
2. 提示条文本最多不能超过 `2` 行。
3. 临时类提示条关闭后，同一提示不建议再次出现；如必须出现，建议最大次数不大于 `3` 次，且三次间隔需超过 `1` 天。
4. 同一页面，同时有且只显示 `1` 个提示条。

## 三、组件分类及交互规则

### 适用平台

手机、折叠屏、平板

### 3.1 信息展示型

根据业务场景选择合适的提示条颜色配置：

| 颜色 | 语义 |
| --- | --- |
| `黄色` | 表示"注意" |
| `红色` | 表示"警示" |
| `蓝色` | 表示"信息" |

**各区域交互说明：**

- `① 左侧文案`（必选）：对提示文案进行描述，提示条文本最多不能超过 `2` 行
- `② 操作按钮`（非必选）：最多支持 `1` 个操作按钮，引导按钮强调性强，支持原地操作和跳转
- `③ 功能图标：关闭/跳转`（非必选）：
  - 有系统级重要信息时不建议关闭
  - 在大屏设备最多展示 `2` 个功能/功能入口
  - 提示条关闭后，同一提示不建议再次出现；如必须出现，建议最大次数不大于 `3` 次，且三次间隔需超过 `1` 天
