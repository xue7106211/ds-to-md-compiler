## 一、概述

搜索框是精确、高效进行搜索查找的组件，比起一个单独的搜索 icon，搜索框是一种目标更大、更易发现、更易触达且能展示更多提示信息的搜索入口。

### 默认态选择

业务可从两种搜索框默认态中选择：

1. **默认显示，上滑隐藏**
2. **默认隐藏，下拉显示**

## 二、组件构成

搜索框由**左、中、右**三部分组合构成：

| 位置 | 序号 | 主要构成 | 说明 |
| --- | --- | --- | --- |
| 左（可选） | ❶ | 返回箭头 | 退出搜索操作 |
| 中 | ❷ | 🔍 搜索 icon | 示意搜索的功能 |
| 中 | ❸ | 文字提示 | 提示用户可搜索的对象 |
| 中 | ❹ | 输入框 | 限定输入的区域 |
| 右（可选） | ❺ | 功能按钮 | 如搜索按钮、扫码图标等 |

## 三、组件位置及交互规则

### 通用模型

#### 默认态 — 仅输入框

- 默认态的搜索框是搜索入口的一种，同样可作为搜索入口的还有搜索图标。
- 已 MIUIX 化：**是**

**文字提示格式**

- 格式为 `搜索+（搜索的对象）`。
- ⚠ 避免将"搜索"这样的词用作占位符文本，因为这类词并不会提供任何有用的信息。

#### 激活态 — 输入框 + 操作按钮

点击默认态之后，起输入法键盘，同时搜索框变为激活态。

**文字规则**

- 部分业务在文字前有搜索标签。
- 输入文字后，原搜索提示消失，以输入的内容替代。
- 光标于输入末尾闪烁。

**清除按钮**

- 点击清除按钮，回到「激活态-输入前」的状态，但不会退出搜索。
- 未输入任何内容时，无清除按钮；输入内容后，出现清除按钮。

**取消 — 文字按钮**

- 点击「取消」退出搜索。
- 已 MIUIX 化：**是**

#### 激活态 — 返回按钮 + 输入框

**返回按钮**

- 点击「返回按钮」退出搜索，等同于「取消」。
- 已 MIUIX 化：**是**

#### 激活态 — 返回按钮 + 输入框 + 搜索按钮

**搜索 — 文字按钮**

- 适用于输入后，需要点击"搜索"才能匹配最终目标项的情况。
- 在搜索框右侧特意添加"搜索"按钮的场景，一般是**非即时搜索**。即输入内容的同时，下方显示的并不是最终的搜索结果条目，而是关联词等，需要再点击一下"搜索"才能执行。
- 还能够满足一些运营需求，比如搭配在激活态时采用预置的搜索词，来显示新品关键词，此时用户无需输入，可直接点击搜索前往。
- 已 MIUIX 化：**暂未**

### 「取消按钮」vs「返回按钮」方案选择指南

这两种方案发挥的作用相同，以下提供几种给设计师考量的因素：

1. 首先保证应用内的一致性、系统应用一致性。
2. 「取消按钮」的方案在动效上能做到"变化幅度更轻盈"。
3. 如果输入框右侧有操作按钮（例如扫码、拍照、下载等），出于减少误触、视觉平衡原因可以选择「左侧返回按钮」的解决方案。

### 其他注意事项

1. 搜索框在输入时，只能**单行显示**文字，搜索框不会为了适应文字而扩宽或增高；若文字超出输入框可显示长度，文字左移，超出部分隐藏。
2. 当同时有搜索框和提示条时，搜索框在上，下滑拉出时，先出现搜索框，再出现提示条。
3. 当应用采用分栏结构时，搜索框**不允许**位于「侧边导航栏」。

```yaml
interaction_rules:
  variants:
    - id: default_input_only
      label: 默认态 — 仅输入框
      miuix: true
      hint_format: "搜索+（搜索的对象）"
    - id: active_input_cancel
      label: 激活态 — 输入框+取消按钮
      miuix: true
      exit_action: tap_cancel_text
    - id: active_back_input
      label: 激活态 — 返回按钮+输入框
      miuix: true
      exit_action: tap_back_icon
    - id: active_back_input_search
      label: 激活态 — 返回按钮+输入框+搜索按钮
      miuix: false
      search_mode: non_instant
  text_behavior:
    single_line_only: true
    overflow: shift_left_and_clip
    clear_button: appears_after_input
  layout_constraints:
    sidebar_nav_placement: forbidden
    coexist_with_toast: search_bar_on_top
```

## 五、特殊情况适配

### 1. 大字号 / 多语言

- 搜索栏的文字（包括搜索框内文字、右侧操作按钮文字）的**最大限制字号**为 `26sp`，一旦达到此值不再跟随系统继续放大。
- 对于**高字体**（目前包括藏语、缅甸语），最大限制字号为 `20sp`。
- 搜索框内提示词仍然放不下的，末尾加 `...`。
- ❗ 当激活态搜索框右侧有操作按钮时，搜索框的宽度会进一步变窄，因此业务设计师需要考虑到多语言/大字号的影响，谨慎在激活态放置长提示词，尽量精简以避免信息缺失。

```json
{
  "large_font_rules": {
    "max_font_size_sp": 26,
    "tall_script_max_font_size_sp": 20,
    "tall_scripts": ["藏语", "缅甸语"],
    "overflow_strategy": "ellipsis",
    "note": "激活态右侧有操作按钮时宽度变窄，需精简提示词"
  }
}
```
