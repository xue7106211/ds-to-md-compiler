## 一、概述

列表（`List`）包含一系列**相同宽度的列表项**，适合连续、多行呈现同类数据（例如图片和文本）。列表按一定规律排序（如按时间、按字母等），列表项可以包含文本、图片以及操作。

### 1. 适用的平台

- 手机
- Fold 折叠屏内屏
- 平板设备

## 二、列表构成

列表由**文本区域**和**头部、尾部区域**组成：

- **文本区域**：主标题和辅助文字（核心呈现，具体见「三、分类及交互规则」）。
- **头部、尾部区域**：图形、功能控件（辅助表达内容、状态和功能延展）。

| 区域 | 必选/可选 | 说明 |
| --- | --- | --- |
| 文本区域 | 通常情况下必选 | 列表的核心呈现。 |
| 头部区域 | 可选 | 列表内容图形化示意，数量最多支持一张图。 |
| 尾部区域 | 可选 | 状态展示或功能表达，可为文本或功能控件。 |

![列表构成：文本区域（主标题+辅助文字）、头部、尾部](image-placeholder-list-structure)

```yaml
visual_data:
  component: List
  regions:
    - id: text
      role: 文本区域
      required: true
      content: 主标题(必选)、辅助文字(可选)
    - id: leading
      role: 头部区域
      required: false
    - id: trailing
      role: 尾部区域
      required: false
  tokens_from_figure: null
```

## 三、分类及交互规则

### 1. 常规型列表项规则

#### 文本区域通用规则

| 要素 | 规则 |
| --- | --- |
| **主标题（必选）** | 保持文本简洁，使内容易于阅读。 |
| **辅助文字（可选）** | 通常是主标题区的补充说明；以短语或句子形式呈现，句末无标点；不与主标题完全重复或部分重复；不提及用户。文案折行限制为三行，超出部分用省略号。 |

#### 尾部区域通用规则

- 对列表进行状态展示或功能表达，可为文本或功能控件。
- 状态文案可为空；或为下一级重要描述的前置。
- 文案折行限制为三行，超出部分用省略号。
- 状态文案建议：不宜过长，应直观表达状态，避免只有动词。
- 区分辅助文字和状态文案：辅助文字侧重描述主标题，状态文案是下一页重要信息前置；状态文案过长时优先优化文案，特殊情况下可放置在辅助文字。

#### 常规型列表项类型与交互

| 列表项类型 | 各区域交互说明 | 最佳实践 | 线上错误案例 |
| --- | --- | --- | --- |
| **展示详情型列表项** | 一般不响应点击，特殊情况下可跳转详情。尾部区域（可选）：当有详细信息需要补充时使用（如时间、数据、价格等）。 | 系统设置-省电与电池-系统桌面-耗电排行；更多设置-卡包-交通卡-更多操作；联系人；页面导航型：系统设置-连接与共享、隐私与安全-照片和视频；联系人-用户名-二维码名片。 | — |
| **页面导航型列表项** | 整体响应对列表内任意区域的点击，进入下一级页面。尾部区域规则见上。 | 隐私与安全-虚拟身份管理。 | — |
| **开关操作列表项** | 整体响应对列表内任意区域的点击，在当前页面执行开关行为。注意设置默认开关状态。 | 屏幕使用时间-设备使用设置。 | 系统设置-通知管理（整体响应开关变成了局部响应开关）。 |
| **多选项切换（近手弹窗）列表项** | 整体响应对列表内任意区域的点击，在近手弹窗进行选项切换。尾部区域：缺省有兜底文案或默认选项；折行限制三行。 | 系统设置-通知与状态栏。 | — |
| **向下展开列表项** | 局部响应对控件区域的点击，向下展开列表项，向上收起。二级展开项头部缩进，遵循常规型与选择型列表项规则。 | 邮件-设置。 | — |
| **添加去除列表项** | 局部响应对控件区域的点击，在当前页面进行列表项增删。 | 日历-设置-全球节日。 | — |
| **清除列表项** | 可为单操作或多操作。单操作：局部响应对控件区域的点击，在当前页面清除该列表项。多操作：整体响应进入下一级页面，尾部区域响应具体操作。 | 手机管家-自动任务。 | — |
| **附加按钮型列表项** | 单操作：局部响应对控件区域的点击进行相关操作。多操作：整体响应进入下一级页面，尾部区域响应具体操作。 | 单操作：问题反馈-日志抓取；多操作：我的设备-新版本；快捷功能-门卡-添加门卡。 | 点击为整体响应、按钮没有作用。 |
| **附加查看详情型列表项** | 多操作列表项。整体响应为主要操作，通常接抽屉或下一级页面；尾部区域响应接下一级页面（详情描述）。 | 通话记录；系统设置-蓝牙。 | — |

### 2. 选择型列表项规则

#### 2.1 选择常规样式（单选）

| 区域 | 规则 |
| --- | --- |
| **文本区域** | 主标题（必选）、辅助文字（可选），规则同常规型。 |
| **尾部区域** | 单选或多选控件。 |
| **交互** | 整体响应对列表内任意区域的点击，进行选中。成组的单选列表中，选中态的选项只支持一个。**不允许不选中任何一个选项**。选中态具有排他性：首次点击未选项立即激活选中态并自动取消其他选项；重复点击已选项保持当前选中态不变化（点击行为无效），需通过选择其他选项进行切换。 |

- 最佳实践：线上暂无。
- 线上错误案例：日历-设置-其他历法；系统设置-隐私与安全-位置-应用名称；邮箱-设置-账户切换；家人守护-通信限制（无底色）。

#### 2.2 选择附加其他功能样式

| 样式 | 说明 |
| --- | --- |
| **头部区域** | 选择控件。 |
| **文本区域** | 主标题（必选）、辅助文字（可选），同常规型。 |
| **尾部区域** | 其他功能控件。 |
| **多选型列表** | 整体响应；支持选中态叠加。点击未选项立即激活选中态不影响其他已选项；点击已选项立即取消当前选中态。头部区域（特殊情况）：列表内容图形化示意，最多一张图。 |
| **带下级入口型单选** | 多操作列表项；整体响应进行选中；尾部区域响应进入下一级页面。不允许不选中任何一个选项；选中态具有排他性（同上）。最佳实践：系统设置-显示与亮度-色彩风格；更多操作-卡包-门卡-更多设置-快捷卡片设置。错误案例：家人守护-通信限制。 |
| **状态启用型列表（蓝牙、WiFi）** | 局部响应对控件区域的点击，进入详情页面。选中态通过背景色表达链接感，不通过选择控件。头部区域（特殊情况）同多选。最佳实践：系统设置-WLAN、蓝牙。 |

#### 2.3 选择型列表分组规则

当页面中有多个单选信息条目时，选择「每条信息独立显示」还是「整张卡片一起显示」可参考：

| 判断维度 | 建议 |
| --- | --- |
| **判断内容是否有相互独立性** | 看选项内容关系：信息相互独立建议选项样式独立；彼此为某一信息子集且并列建议用整张卡片。选项为有限集合且并列（如选择不同时间、周一到周日、手动或自动） vs 选项无限添加且相互独立（如选择不同模式、不同类型）。 |
| **选择后是否有选项的二次展开** | 当有选项选中时有附加信息时，推荐使用独立卡片列表。 |
| **是否有辅助描述信息** | 当选项内有次要文字时，推荐使用独立卡片列表。 |

#### 2.4 选择型列表对勾位置

- **整体原则**：在 Xiaomi HyperOS 系统内的单选列表中，**所有勾选项默认在右侧**。仅当列表内右侧有独立点击区域且该列表可选中时，该页面内所有单选列表的勾选项**避让到左侧**展示；该规则也覆盖系统中菜单、近手菜单等有同样选中表达的控件。

| 对勾在左侧 | 对勾在右侧 |
| --- | --- |
| 当页面内存在右侧独立点击区域且列表可选中时 | 默认 |

#### 2.5 选择型列表底色规则

| 分类 | 描述 | 样式规则 | 典型案例 |
| --- | --- | --- | --- |
| **即时生效** | 点击选择后立即生效，数据实时同步。 | 单选：不带轮廓底色，选中项有对勾且文字变蓝。多选：不带轮廓底色，选中项有对勾，文字不变色。 | — |
| **确认后生效** | 表达临时选中状态、等待执行中，点击确认/提交等按钮才生效，支持批量处理。 | 单选：带轮廓底色，选中项有对勾且文字变蓝。多选：带轮廓底色，选中项有对勾，文字不变色。 | — |

### 3. 列表项管理规则

#### 3.1 列表排序

| 类型 | 说明 |
| --- | --- |
| **单一排序** | 点按排序控件或长按列表项进行拖拽。 |
| **排序复合其他操作** | 点按排序控件或长按列表项进行拖拽，尾部支持多选、开关、增加删除等控件。 |

#### 3.2 列表滑动删除

列表项如有删除需求，**左滑列表项**进行删除。

#### 3.3 列表分组标题

- 若有可选择的多个标题，使用带有展开符号的标题；若无则使用纯文字标题。
- 多标题内容示例：尺寸、时间、价格等维度。

#### 3.4 列表分割线

| 分割线类型 | 描述 | 典型场景 |
| --- | --- | --- |
| **套卡分割线** | 用于套卡列表，区分同个卡内内容。 | 自定义/开关展开时用作展开前后区分（如护眼功能设置）；大图与列表之间区分（如用眼时长统计）。 |
| **不套卡分割线** | 用于通栏，区分不同内容块。 | 例如文件管理相关页面。 |

### 5. 响应式规则

#### 5.1 不同设备列表扩展

列表在不同设备间流转，通过调整**【中间区域】宽度**增加内容承载力，以及调整其左右边距以更好适应 Pad 等大屏幕。

| 设备 | 手机端 | 折叠屏 | Pad 端 |
| --- | --- | --- | --- |
| 示意图/扩展区域对比 | （见源 PDF 第十四页） | — | — |

#### 5.2 内容超长处理

1. 优先保证左侧文本完整，左侧文本不能无内容。
2. 主标题和辅助文字折行无行数限制，具体跟随业务实际使用情况调整。
3. 下级页面前置的文案折行限制为**三行**，超出部分用省略号（例如「文案+进入二级」「文案+近手弹窗」）。

#### 5.3 文本宽度

| 文本情况 | 规则/示意图 |
| --- | --- |
| 左侧多、右侧少 | 右侧最小值为 `52dp`。左侧区域新增图标场景：左侧区域(图标)+中间区域(标题内容&辅助文案)=`264`。 |
| 左右侧均多 | `200dp`：`116dp`（右侧区域最大值）。左侧区域新增图标场景：左侧区域(图标)+中间区域(标题内容&辅助文案)=`200`。 |

```yaml
box_model:
  text_width:
    left_heavy_min_right_dp: 52
    left_heavy_total_dp: 264
    both_heavy_total_dp: 200
    right_max_dp: 116
```

## 五、特殊情况适配

### 1. 多语言

- 参考：**多语言设计指南**。
- 建议在任何情况下保证主要文字的完整显示，文本较长时至少折三行。
- 同一行内图片/图标+文字控件数量建议按下方表格控制：同一年内文字控件数量建议不超过 2 个；超过 2 个时建议限制文案为「已开启」「已关闭」等较确定的文案。
- 尽量不使用跑马灯；如需使用，同一界面建议不超过 3 个。

## 七、其他注意事项

### 文案规范

1. 保持文本简洁，使内容易于阅读；主标题清晰、直观并且勿添加结尾标点符号。
2. 文案优先级：尽量使用主标题，不用辅助文字，一次性把话说清楚；当主标题无法描述清楚或需要有重要细节补充时再使用辅助文字。
3. 区分辅助文字和尾部控件文案：辅助文字用于解释主标题，尾部操作识别区文案用于将下一级的重要信息前置，两者作用不要混淆。
