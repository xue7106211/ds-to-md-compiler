## 一、概述

近手菜单是快速在近手处唤起若干子级选项的轻量级交互菜单，这些子选项通常互斥，点击选项后不可以跳转新的页面。

### 1. 适用的平台

- 手机
- Fold 折叠屏内屏
- 平板设备

## 二、组件构成

| 构成项 | 说明 |
| --- | --- |
| ① 选项标题 | 主标题文字 |
| ② 选项副标题 | MIUIX 提供副标题接口给业务使用，业务可根据需求自行调用 |
| ③ 选项 icon | 同一近手菜单内各选项要么都有 icon，要么都没有 icon，保证样式一致。MIUIX 提供 icon 接口，选项文字颜色与 icon 颜色必须一致，业务不可随意更改 |
| ④ ✔ 和选中态色彩 | ✔ 和底色统一采用系统蓝色，不开放给业务自定义 |

## 三、组件分类及交互规则

### 终端

手机 / 折叠 / Pad

### 类型 1：普通近手菜单 - 单行

| 规则 | 说明 |
| --- | --- |
| 标题文字 | 最多支持展示一行 |
| 宽度 | 最大 `288dp`，最小 `200dp`，跟随内容自适应 |
| 选中态 | 背景变蓝，文字变蓝，右侧打勾 |

### 类型 2：普通近手菜单 - 双行

| 规则 | 说明 |
| --- | --- |
| 标题文字 | 最多支持展示一行 |
| 描述信息（可选） | 最多支持展示两行 |
| 宽度 | 最大 `288dp`，最小 `200dp`，跟随内容自适应 |

### 类型 3：展示 icon 的近手菜单

| 规则 | 说明 |
| --- | --- |
| icon 使用 | 业务可从辨识度、装饰性和易用性提升角度出发增加 icon |
| 选中态 | 背景变蓝，文字与图标统一变蓝，右侧打勾 |
| 一致性 | 同一近手菜单内各选项要么都有 icon，要么都没有 icon |
| 图标来源 | 优先调用图标库图标，图标库无法满足时支持自定义图标（需按图标规范输出） |

### 通用交互规则

| 规则 | 说明 |
| --- | --- |
| 触发点选即消失 | 近手菜单触发点选操作时即会消失 |
| 点击空白关闭 | 点击空白处关闭下拉菜单 |
| 列表选中态 | 当近手菜单从列表中触发时，列表中相应条目有选中态效果：a. 整行响应的整行触发置灰；b. 独立区域响应的独立区域置灰 |
| 不支持多选 | 近手菜单不支持多选 |

### 4.4 热区

| 触发方式 | 说明 |
| --- | --- |
| 全面积触发 | 近手菜单通常与列表搭配使用，触发区域为当前列表条目的全部面积 |

## 六、易混淆组件辨析

### 下拉菜单 vs 近手菜单

| 维度 | 下拉菜单 | 近手菜单 |
| --- | --- | --- |
| 功能定位 | 对平级应用功能的收纳 | 仅针对当前项的操作 |
| 出现位置 | 工具栏/标题栏通常由更多图标触发；长按列表 | 点击带有更多操作的列表（及分组标题）触发 |
| 功能包含 | 普通列表 / 多选一列表 / 进入二级列表 / 支持分组 | 普通列表 / 支持分组 |
