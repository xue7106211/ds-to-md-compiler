## 一、概述

菜单是通过点击固定位置的下拉按钮，展开子层级菜单的这一类控件的总称，点击展开菜单的选项后可以跳转新的页面。满足业务对若干平级功能的收纳诉求，悬浮菜单是模态的。

## 二、组件构成

| 类型 | 主要构成 | 区域概述 |
| --- | --- | --- |
| 横排 | ① 图标 + ② 文字（可选） | 选项由图标和文字组成 |
| 竖排 | ① 图标（可选） + ② 文字 + ③ 右侧小图标 + ④ 分割线 | 支持图标、文字、右侧箭头、分割线等元素 |

## 三、组件分类及交互规则

### 3.1 手机 / 折叠 / Pad

#### 横排

| 子类型 | 规则 |
| --- | --- |
| 仅图标 | 1. 仅显示图标时需保证图标的优秀可识别性<br>2. 最多显示三个选项<br>3. 优先调用图标库图标，图标库无法满足时支持自定义图标（需按照图标规范输出） |
| 图标+文字 | 1. 最多显示三个选项<br>2. 需注意文字折行问题，及在多语言下的视觉表现 |

#### 竖排

| 子类型 | 规则 |
| --- | --- |
| 普通操作列表 | 文字折行：目前规范折两行 |
| 二级列表 | 二级样式需要进一步确认。点击二级列表进入二级页面。二级页面内支持普通操作列表和多选一列表。二级页面内不支持继续放置二级列表。 |
| 多选一列表 | 列表被选中时：① 文字（及图标）颜色变为 `#3482FF`（系统蓝）；② 右侧显示选中对勾标记；③ 底色不做改变。未选中列表样式与普通操作列表一致。仅支持多选一，不支持多选。 |
| 组合样式 | a. 普通操作列表<br>b. 二级列表<br>c. 多选一列表<br>d. 底部单独分割（不再使用分离样式，分离样式由底部分割代替） |
| 使用分割线的情况 | 1. 不同类型的列表之间需要进行分割<br>2. 需要着重强调的列表功能可以位于底部分割（例如多选/设置）<br>3. 功能分组时也有必要分割 |
| 图标和文字 | 竖排菜单支持显示图标。注意选项样式需保持一致，不能有些选项有图标，有些选项没有图标。图标与文字整体水平居中对齐。 |

### 3.2 交互规则（通用）

| 规则 | 说明 |
| --- | --- |
| 菜单触发点选操作时即会消失 | 普通操作列表点击后菜单关闭 |
| 点击二级列表进入二级页面 | 进入二级页面 |
| 点击多选一列表会触发选择操作，菜单不会消失 | 多选一列表为选择态切换 |
| 点击空白处关闭菜单 | 关闭菜单 |
| 菜单触发时背景叠加 | 30% `#000000` |

## 六、易混淆组件辨析

### 6.1 菜单与近手菜单

| 对比项 | 菜单 | 近手菜单 |
| --- | --- | --- |
| 作用 | 对平级应用功能的收纳 | 仅针对当前项的操作 |
| 出现位置 | 工具栏/标题栏通常由更多图标触发 | 长按列表或点击带有更多操作的列表（及分组标题）触发 |
| 功能 | 包含普通列表/多选一列表/进入二级列表/支持分组 | 包含普通列表/支持分组 |

### 6.2 菜单横排与竖排如何选择

| 类型 | 适用场景 |
| --- | --- |
| 横排 | 仅用于与图标相关的操作 |
| 竖排 | 更加常用的样式 |
